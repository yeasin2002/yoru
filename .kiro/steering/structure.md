# Project Structure

## Directory Organization

```
openkit/
├── src/                  # Source code (TypeScript)
│   ├── cli.ts           # Commander.js CLI entrypoint
│   ├── pipeline.ts      # Core pipeline runner
│   ├── parser.ts        # OpenAPI parsing and dereferencing
│   ├── filter.ts        # Filtering engine
│   ├── config.ts        # Configuration file loader
│   ├── transformers/    # Output format transformers
│   │   ├── markdown.ts  # Markdown documentation
│   │   ├── ts-types.ts  # TypeScript types
│   │   ├── zod.ts       # Zod schemas
│   │   └── sdk.ts       # SDK generation
│   └── utils/           # Shared utilities
├── test/                # Test files (Vitest)
├── bin/                 # CLI executable
│   └── openkit.js       # Shebang entrypoint
├── dist/                # Build output (generated, gitignored)
├── .changeset/          # Changeset configuration
├── .github/             # GitHub Actions workflows
├── .husky/              # Git hooks
└── node_modules/        # Dependencies (gitignored)
```

## Source Code (`src/`)

- **Entry Point**: `src/index.ts` - Main export file that re-exports all public APIs
- **CLI**: `src/cli.ts` - Commander.js CLI implementation
- **Pipeline**: `src/pipeline.ts` - Core transformation pipeline
- **Transformers**: `src/transformers/` - Output format implementations
- **Pattern**: Export all public APIs through `index.ts` using `export * from './module'`

## Transformers (`src/transformers/`)

Each transformer follows the contract:

```typescript
type Transformer<TOptions = Record<string, unknown>> = (
  spec: OpenAPIObject,  // already parsed + dereferenced
  options?: TOptions
) => Promise<string | Record<string, string>>
```

- **Markdown**: `@scalar/openapi-to-markdown` integration
- **TypeScript**: `openapi-typescript` for zero-runtime types
- **Zod**: `openapi-zod-client` for runtime validation schemas
- **SDK**: `@hey-api/openapi-ts` for full client generation

## Tests (`test/`)

- **Naming**: `*.test.ts` files matching source file names
- **Imports**: Import from `../src/module` (not from dist)
- **Framework**: Vitest with globals enabled (no need to import `describe`, `it`, etc.)
- **Focus**: Transformer outputs, filtering logic, CLI commands

## Build Output (`dist/`)

Generated by tsdown, contains:

- `index.cjs` - CommonJS bundle
- `index.js` - ESM bundle
- `index.d.ts` - TypeScript declarations
- Source maps for all files

## Configuration Files (Root)

- `package.json` - Package metadata and scripts
- `openkit.config.ts` - Example configuration file
- `tsconfig.json` - TypeScript compiler options
- `tsdown.config.ts` - Build configuration
- `vitest.config.ts` - Test configuration
- `eslint.config.js` - Linting rules
- `.prettierrc` - Code formatting rules
- `.lintstagedrc.json` - Pre-commit hook configuration

## Package Exports

The package uses conditional exports:

```json
{
  ".": {
    "import": "./dist/index.js", // ESM
    "default": "./dist/index.cjs" // CJS fallback
  },
  "./bin": "./bin/openkit.js"
}
```

## CLI Structure

```
bin/
└── openkit.js          # Shebang executable that imports src/cli.ts
```

## Files Included in NPM Package

- `dist/` - Build output
- `bin/` - CLI executable
- `README.md` - Documentation
- `LICENSE` - License file

Excluded via `.npmignore`:
- `src/` - Source files
- `test/` - Test files
- Development configuration files
